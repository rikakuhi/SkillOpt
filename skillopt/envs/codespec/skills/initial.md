# Codespec Skill

## Requirement Analysis
- Read `final.md` carefully and decompose it into discrete functional requirements.
- Identify the core problem domain, constraints, and acceptance criteria before designing.
- Pay attention to implicit requirements (error handling, edge cases, performance).

## Design Document Quality
- Each feature should be described with enough detail that a developer could implement it without ambiguity.
- Group related features logically and use clear markdown headings for each section.
- Include input/output specifications, data flow, and key algorithms where applicable.

## Completeness Discipline
- Ensure every requirement from the specification is mapped to at least one design section.
- Do not skip error handling, validation, or boundary conditions.
- Cross-reference related features to show dependencies and interactions.

## Precision Discipline
- Avoid vague or generic descriptions that could apply to any project.
- Use specific variable names, function signatures, and data structures where possible.
- State preconditions, postconditions, and invariants for each feature.
